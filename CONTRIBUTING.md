# 🤝 Как внести вклад в Percepta

Спасибо, что хотите помочь проекту! Ваш вклад важен для нас.

---

## 🐛 Сообщение об ошибке

Если вы нашли ошибку, пожалуйста, создайте Issue в GitHub:

1. Проверьте, что ошибка не была уже описана в [Issues](https://github.com/msernv/percepta/issues)
2. Создайте новый Issue с пометкой `bug`
3. В описании укажите:
   - **Шаги для воспроизведения:** что нужно сделать, чтобы увидеть ошибку
   - **Ожидаемое поведение:** что должно было произойти
   - **Скриншоты:** если возможно, добавьте изображения
   - **Окружение:** ваша ОС, браузер, версия Python

---

## 💡 Предложение новой фичи

Если у вас есть идея, как улучшить Percepta:

1. Проверьте, что фича не была уже предложена в Issues
2. Создайте новый Issue с пометкой `enhancement`
3. Опишите:
   - **Проблему:** какую проблему решает фича
   - **Решение:** как вы видите реализацию
   - **Примеры:** если есть, покажите, как это будет работать

---

## 🛠 Pull Request (Как предложить изменения)

Если вы хотите добавить код в проект:

1. **Форкните репозиторий** — нажмите кнопку Fork на GitHub.
2. **Клонируйте свой форк** — скачайте код к себе на компьютер.
3. **Создайте ветку для фичи**:
   ```bash
   git checkout -b feature/amazing-feature
Внесите изменения — напишите код.

Закоммитьте изменения:

bash
git add .
git commit -m "Добавлена новая фича: ..."
Запушьте ветку:

bash
git push origin feature/amazing-feature
Откройте Pull Request — на GitHub нажмите "Compare & pull request".

✅ Требования к Pull Request
Код должен быть чистым и понятным.

Добавьте комментарии к сложным местам.

Если вы добавляете новую метрику, обновите документацию.

Убедитесь, что проект запускается без ошибок.
